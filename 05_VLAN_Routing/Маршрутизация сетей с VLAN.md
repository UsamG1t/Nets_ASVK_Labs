## Построение сети с VLAN

Для изучения маршрутизации в сетях с VLAN разберём топологию с двумя VLAN-сетями и настроим трафик между ними через пограничный маршрутизатор, способный передавать пакеты в обеих сетях.

![](Attached_materials/05_VlanRouting.png)

Для работы создайте 4 [клона](../01_FirstStart/Настройка%20системы%20для%20выполнения%20лабораторных.md) согласно топологии сети. Для создания соединений между машинами необходимо в VirtualBox настроить сетевые интерфейсы (описание настройки подключения находится в разделе [настройки сетевых подключений](../02_SystemGreetings/Знакомство%20с%20системой.md#работа-с-сетевыми-интерфейсами)):

 + `R1`:
	 + Adapter2 — left4
 + `R2`:
	 + Adapter2 — trunk
 + `R3`:
	 + Adapter2 — right8
 + `S1`:
	 + Adapter2 — left4
	 + Adapter3 — trunk
	 + Adapter4 — right8

### Настройка абонентов

:round_pushpin: 1. С помощью команд управения интерфейсами и настройки IP-адресов на R1 и R3 включите интерфейсы и установите на них IP-адреса согласно топологии:

`@R1`
```console
[root@R1 ~]# ip link set eth1 up
[root@R1 ~]# ip addr add dev eth1 10.0.4.1/24
[root@R1 ~]#
```

`@R3`
```console
[root@R3 ~]# ip link set eth1 up
[root@R3 ~]# ip addr add dev eth1 10.0.8.1/24
[root@R3 ~]#
```

### Настройка пограничного маршрутизатора

:information_source: Поскольку трафик необходимо будет направлять через пограничный маршрутизатор, на нём должны быть явные интерфейсы локальных сетей с присвоенными IP-адресами. Так как физический канал связи по топологии один, необходимо создать виртуальные интерфейсы, каждый из которых будет направлен в свою VLAN-сеть.

:round_pushpin: 1. С помощью _команды создания связанных виртуальных интерфейсов_ :new: на R2 создайте два виртуальных VLAN-интерфейса из VLAN-сетей согласно топологии

`@R2`
```console
[root@R2 ~]# ip link add link eth1 name vlan4 type vlan id 4
[root@R2 ~]# ip link add link eth1 name vlan8 type vlan id 8
[root@R2 ~]#
```

:round_pushpin: 2. С помощью команд управения интерфейсами и настройки IP-адресов на R2 включите интерфейсы и установите на них IP-адреса согласно топологии:

`@R2`
```console
[root@R2 ~]# ip addr add dev vlan4 10.0.4.254/24
[root@R2 ~]# ip addr add dev vlan8 10.0.8.254/24
[root@R2 ~]# ip link set eth1 up
[root@R2 ~]# ip link set vlan4 up
[root@R2 ~]# ip link set vlan8 up
[root@R2 ~]#
```

:information_source: Для передачи пакетов между сетевыми интерфейсами необходимо изменить настройку `net.ipv4.conf.all.forwarding`:

:round_pushpin: 3. С помощью команды _перечачи пакетов между интерфейсами_ :new: `sysctl net.ipv4.conf.all.forwarding=1` разрешите передачу пакетов на R2

`@R2`
```console
[root@R2 ~]# sysctl net.ipv4.conf.all.forwarding
net.ipv4.conf.all.forwarding = 0
[root@R2 ~]# sysctl net.ipv4.conf.all.forwarding=1
[root@R2 ~]#
```

### Настройка коммутатора

:round_pushpin: 1. С помощью команд управления интерфейсами создайте на S1 интерфейс типа bridge с указанием параметра vlan_filtering и свяжите с ним все используемые физические интерфейсы

`@S1`
```console
[root@S1 ~]# ip link add dev br0 type bridge vlan_filtering 1
[root@S1 ~]# ip link set eth1 master br0
[root@S1 ~]# ip link set eth2 master br0
[root@S1 ~]# ip link set eth3 master br0
```

:round_pushpin: 2. С помощью команд настройки VLAN установите на S1 VLAN на интерфейсы согласно топологии

`@S1`
```console
[root@S1 ~]# bridge vlan add vid 4 dev eth1 pvid untagged
[root@S1 ~]# bridge vlan add vid 8 dev eth3 pvid untagged
[root@S1 ~]# bridge vlan add vid 4 dev eth2
[root@S1 ~]# bridge vlan add vid 8 dev eth2
```

:round_pushpin: 3. C помощью команд управления интерфейсами включите на S1 все используемые интерфейсы

`@S1`
```console
[root@S1 ~]# for I in `ls /sys/class/net`; do ip link set $I up; done
```

### Настройка статических маршрутов

:round_pushpin: 1. С помощью команды управления таблицами маршрутизации на R1 и R2 добавьте информацию о маршрутах по умолчанию

`@R1`

[root@R1 ~]# ip route add default via 10.0.4.254

`@R3`

## Самостоятельная работа

![](Attached_materials/05_VlanRouting_work.png)

:round_pushpin: 1. Определить свой вариант **X**, как свой номер по порядку в упорядоченном по алфавиту списке группы.

:round_pushpin: 2. Создать топологию, изображенную выше.

:round_pushpin: 3. `PC1` и `PC3` должны быть объединены в один VLAN с номером (**X** + 10). `PC2` и `PC4` должны быть объединены в другой VLAN с номером (**X** + 20).

:round_pushpin: 4. Реализовать пересылку между VLAN через маршрутизатор `R1`

:round_pushpin: 5. IP-адреса для сетевых устройств в VLAN (**X** + 10) брать из сети 10.0.{**X**+10}.0/24. IP-адреса для сетевых устройств в VLAN (**X** + 20) брать из сети 10.0.{**X**+20}.0/24.

:round_pushpin: 6. Запустить [отчёты](../02_SystemGreetings/Знакомство%20с%20системой.md#Сдача-самостоятельных-работ) на каждой машине и выполнить соответствующие команды:

:warning: ***Важно:*** Поскольку для устройств не зафиксированы номера хостов, их выбор оставляется за исполнителем. При описании команд отчётов используются номера 1, 2, 3, 4 для IP-адресов абонентов и 254 для обозначения IP-адресов маршрутизатора. Необходимо на этом месте указывать выбранные IP-адреса.

 + `report 9 pc1`
	 + ip a show `<PC1→S1 interface>`
	 + ip route
	 + ping -fc3 10.0.{**X**+10}.254
	 + ping -fc3 10.0.{**X**+20}.254
	 + ping -fc3 10.0.{**X**+20}.2
	 + ping -fc3 10.0.{**X**+10}.3
	 + ping -fc3 10.0.{**X**+20}.4
	 + traceroute 10.0.{**X**+10}.3
 + `report 9 pc2`
	 + ip a show `<PC2→S1 interface>`
	 + ip route
	 + ping -fc3 10.0.{**X**+20}.254
	 + ping -fc3 10.0.{**X**+10}.254
	 + ping -fc3 10.0.{**X**+10}.1
	 + ping -fc3 10.0.{**X**+10}.3
	 + ping -fc3 10.0.{**X**+20}.4
	 + traceroute 10.0.{**X**+20}.4
 + `report 9 pc3`
	 + ip a show `<PC3→S2 interface>`
	 + ip route
	 + ping -fc3 10.0.{**X**+10}.254
	 + ping -fc3 10.0.{**X**+20}.254
	 + ping -fc3 10.0.{**X**+10}.1
	 + ping -fc3 10.0.{**X**+20}.2
	 + ping -fc3 10.0.{**X**+20}.4
	 + traceroute 10.0.{**X**+10}.1
 + `report 9 pc4`
	 + ip a show `<PC4→S2 interface>`
	 + ip route
	 + ping -fc3 10.0.{**X**+20}.254
	 + ping -fc3 10.0.{**X**+10}.254
	 + ping -fc3 10.0.{**X**+10}.1
	 + ping -fc3 10.0.{**X**+20}.2
	 + ping -fc3 10.0.{**X**+10}.3
	 + traceroute 10.0.{**X**+20}.2
 + `report 9 r1`
	 + ip a show `<R1→S2 (vlan X+10) interface>`
	 + ip a show `<R1→S2 (vlan X+20) interface>`
	 + ip route
	 + sysctl net.ipv4.conf.all.forwarding
 + `report 9 s1`
	 + bridge vlan show
 + `report 9 s2`
	 + bridge vlan show


:round_pushpin: Полученные отчёты `report.09.pc1`, `report.09.pc2`, `report.09.pc3`, `report.09.pc4`, `report.09.r1`, `report.09.s1`, `report.09.s2` через последовательный порт перенести из виртуальной машины и прислать их преподавателю с подписью выполненного варианта.
